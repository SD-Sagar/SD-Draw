SD-DRAW - DOCUMENTATION
=======================

Project Overview:
-----------------
SD-Draw is a professional-grade, MERN-stack (MongoDB, Express, React, Node.js) infinite vector canvas application. It features a sleek, MS Paint-inspired floating toolbox with a dynamic dark-mode user interface. 

Architecture:
-------------
- Frontend: React + Vite
- Canvas Engine: React-Konva (HTML5 Canvas wrapper for scalable vector graphics)
- State Management: Zustand (handles global tool states, history stack, and active elements)
- Backend: Node.js + Express
- Database: MongoDB (Mongoose ORM)
- Authentication: JWT (JSON Web Tokens) + Bcryptjs

Key Features:
-------------
1. Infinite Canvas Engine:
   - Middle-click OR Hold Ctrl + Left Click to pan infinitely (shows hand-grab cursor).
   - Scroll wheel to zoom in and out smoothly directly to the mouse cursor position.
2. Vector Drawing Tools:
   - Freehand Pencil and Eraser tools (with dynamic stroke thickness).
   - Geometric Shapes: Line, Arrow, Rectangle, and Circle.
3. Advanced Text Tool:
   - Click anywhere to open a dynamically resizing HTML text input box.
   - Text is seamlessly converted into vector graphics upon saving.
   - Double-click (or single-click while the Text tool is active) to instantly re-open and edit existing text.
4. User Authentication & Persistence:
   - Full JWT-based login and registration system.
   - Canvases are uniquely tied to your specific user account. 
   - Backend includes a powerful "in-memory" fallback mechanism allowing the app to run without MongoDB (data rests in RAM until restart).
5. Real-time Collaboration:
   - Peer-to-peer canvas synchronization powered by PeerJS.
   - Low-latency data exchange for elements, transformations, and deletions.
   - Secure Guest-mode support for rapid entry.
6. State Control:
   - Comprehensive Undo/Redo stack (Ctrl+Z, Ctrl+Y).
   - "Delete" or "Backspace" selected elements instantly.
7. Advanced Eraser Logic:
   - Object Eraser: Fast, swipe-based deletion of entire vector shapes.
   - Precision Eraser: Real-time geometric splitting of pencil lines into smaller segments.
8. Export:
   - Save the active viewport directly to a high-resolution PNG.
9. Branding & Licensing:
   - Signature "Sagar Dey" trademark footer in authentication and dashboard.
   - Custom license: Free for personal use; PAID license required for commercial/business use.
   - Resale and removal of branding are strictly prohibited.

Directory Structure:
--------------------
/backend
   /models
      Canvas.js - Mongoose schema mapping arrays of vector elements to user IDs.
      User.js - Secure authentication schema.
   server.js - Express server and JWT API endpoints.
/frontend
   /src
      /components
         AuthPage.jsx - Login/Registration logic and animations.
         CanvasEngine.jsx - The React-Konva core and event listener hub.
         Toolbox.jsx - The floating sidebar UI.
      /store
         useCanvasStore.js - Zustand store for elements and history.
      App.jsx - React Router routing rules.

DEPLOYMENT INSTRUCTIONS
=======================

Prerequisites for Deployment:
-----------------------------
- Node.js (v16 or higher)
- MongoDB Atlas account (for cloud database) OR local MongoDB
- A hosting provider account (Heroku, Railway, AWS, Vercel, etc.)
- Git repository (GitHub, GitLab, Bitbucket)

Option 1: Deploy on Railway.app (Recommended - Simple & Free Tier Available)
---------------------------------------------------------------------------
1. Create a Railway.app account and connect your GitHub repository.
2. Create a new project and select "Deploy from GitHub".
3. Create a MongoDB service in Railway dashboard.
4. Add environment variables in Railway:
   - PORT=5000
   - JWT_SECRET=your_super_secret_key
   - MONGO_URI=<MongoDB connection string from Railway>
   - NODE_ENV=production
5. Build configuration (Railway auto-detects):
   - Root directory: /
   - Build command: npm install --prefix backend && npm install --prefix frontend && npm run build --prefix frontend
   - Start command: cd backend && npm start
6. Deploy and Railway will automatically handle restarts and scaling.

Option 2: Deploy Frontend on Vercel + Backend on Railway
---------------------------------------------------------
Frontend on Vercel:
1. Push your /frontend folder to a GitHub repository.
2. Connect GitHub to Vercel and select the repository.
3. Set build settings:
   - Framework: Vite
   - Build command: npm run build
   - Output directory: dist
4. Add environment variable:
   - VITE_API_URL=https://your-backend-domain.com/api
5. Deploy.

Backend on Railway:
(Follow Option 1 steps for backend deployment)

Option 3: Deploy with Docker & Docker Compose (For Any Cloud Platform)
------------------------------------------------------------------------
1. Create Dockerfile in project root:
   FROM node:18-alpine
   WORKDIR /app
   
   # Install and build frontend
   COPY frontend/package*.json frontend/
   RUN cd frontend && npm install && npm run build
   
   # Setup backend
   COPY backend/package*.json backend/
   RUN cd backend && npm install
   
   # Copy all files
   COPY . .
   
   # Expose port
   EXPOSE 5000
   
   # Start backend (frontend is served from public folder)
   CMD ["node", "backend/server.js"]

2. Update backend server.js to serve frontend build:
   app.use(express.static(path.join(__dirname, '../frontend/dist')));
   app.get('*', (req, res) => {
     res.sendFile(path.join(__dirname, '../frontend/dist/index.html'));
   });

3. Build and run:
   docker build -t sddraw:latest .
   docker run -p 5000:5000 \
     -e MONGO_URI="your_mongodb_uri" \
     -e JWT_SECRET="your_secret" \
     sddraw:latest

4. Push to container registry (Docker Hub, AWS ECR, etc.) and deploy.

Option 4: Deploy on Heroku (Legacy - Older but Stable)
-------------------------------------------------------
1. Install Heroku CLI and login:
   heroku login
   
2. Create a Heroku app:
   heroku create your-app-name
   
3. Add MongoDB Atlas connection:
   heroku config:set MONGO_URI="your_mongodb_uri"
   heroku config:set JWT_SECRET="your_secret_key"
   
4. Create Procfile in root directory:
   web: cd backend && npm start

5. Build frontend before pushing:
   cd frontend && npm run build && cd ..

6. Push to Heroku:
   git push heroku main

Option 5: Deploy on AWS (EC2 + RDS/DocumentDB)
-----------------------------------------------
1. Launch an EC2 instance (Ubuntu 20.04 LTS recommended).
2. Connect via SSH and install Node.js:
   curl -fsSL https://deb.nodesource.com/setup_18.x | sudo -E bash -
   sudo apt-get install -y nodejs

3. Clone your GitHub repository:
   git clone https://github.com/yourusername/SD-Draw.git

4. Setup backend:
   cd SD-Draw/backend
   npm install
   create .env file with MONGO_URI and JWT_SECRET
   npm install -g pm2
   pm2 start server.js --name "sddraw-backend"

5. Build and serve frontend:
   cd ../frontend
   npm install && npm run build
   npm install -g serve
   serve -s dist -l 3000

6. Setup MongoDB:
   - Use AWS DocumentDB or MongoDB Atlas for managed database
   - Update MONGO_URI in backend .env

7. Setup Nginx as reverse proxy (optional but recommended):
   sudo apt-get install nginx
   Configure /etc/nginx/sites-available/default to proxy to ports 5000 (backend) and 3000 (frontend)

8. Setup SSL with Let's Encrypt:
   sudo apt-get install certbot python3-certbot-nginx
   sudo certbot --nginx -d yourdomain.com

Option 6: Deploy on Render.com (Easy & Fast - Free Tier Available)
-------------------------------------------------------------------
Render is a modern cloud platform that makes deployment simple. You'll deploy the backend as a Web Service and frontend as a Static Site.

STEP 1: Prepare Backend for Render
1. Update backend/server.js to serve frontend static files (add at end of routes, before app.listen):
   
   // Serve frontend static files
   const path = require('path');
   app.use(express.static(path.join(__dirname, '../frontend/dist')));
   
   // Catch-all for SPA routing
   app.get('*', (req, res) => {
     res.sendFile(path.join(__dirname, '../frontend/dist/index.html'));
   });

2. Update backend/package.json to include build script:
   "scripts": {
     "start": "node server.js",
     "build": "cd ../frontend && npm install && npm run build && cd ../backend"
   }

STEP 2: Push to GitHub
1. Create a GitHub repository and push your code:
   git init
   git add .
   git commit -m "Initial commit"
   git remote add origin https://github.com/yourusername/SD-Draw.git
   git push -u origin main

STEP 3: Deploy Backend on Render
1. Create a Render.com account (free signup at render.com)
2. Connect your GitHub account to Render
3. Click "New +" and select "Web Service"
4. Connect your SD-Draw repository and select the main branch
5. Configure the Web Service:
   - Name: sddraw-backend (or your preferred name)
   - Environment: Node
   - Build Command: npm install --prefix backend && npm install --prefix frontend && npm run build --prefix frontend
   - Start Command: cd backend && npm start
   - Instance Type: Free (or Starter for production)
6. Add environment variables under "Environment":
   - PORT = 5000 (Render assigns this automatically, but set it explicitly)
   - NODE_ENV = production
   - JWT_SECRET = your_super_secret_key
   - MONGO_URI = mongodb+srv://username:password@cluster.mongodb.net/sddraw?retryWrites=true&w=majority
   - FRONTEND_URL = https://your-sddraw-frontend.onrender.com (set after frontend is deployed)
7. Click "Create Web Service" and Render will deploy automatically
8. Note your backend URL (e.g., https://sddraw-backend.onrender.com)

STEP 4: Deploy Frontend on Render (Optional - Alternative to serving from backend)
If you prefer to serve frontend separately as a Static Site:
1. Click "New +" on Render dashboard and select "Static Site"
2. Connect the same repository
3. Configure the Static Site:
   - Name: sddraw-frontend
   - Build Command: npm install --prefix frontend && npm run build --prefix frontend
   - Publish Directory: frontend/dist
4. Add environment variable:
   - VITE_API_URL = https://sddraw-backend.onrender.com/api
5. Click "Create Static Site" and deploy

Note: If you deploy frontend separately, ensure CORS is enabled in backend/server.js:
   const cors = require('cors');
   app.use(cors({
     origin: 'https://your-sddraw-frontend.onrender.com',
     credentials: true
   }));

STEP 5: Connect MongoDB
1. Create a MongoDB Atlas account (free tier available at mongodb.com/cloud/atlas)
2. Create a cluster and get your connection string
3. Update MONGO_URI in Render environment variables with the MongoDB Atlas URI

STEP 6: Manual Deploy Triggers
- Render auto-deploys on every push to main branch
- To manually trigger: Go to Render dashboard > Select service > Click "Manual Deploy"

STEP 7: Monitor & Troubleshoot
- View logs: Dashboard > Service > Logs
- Check deployment status: Dashboard > Service > Events
- For database issues, ensure your MongoDB Atlas IP whitelist includes Render's IP range (add 0.0.0.0/0 for testing, then restrict in production)

TIPS FOR RENDER DEPLOYMENT:
- The free tier includes 750 hours/month per service (sufficient for single-user/testing)
- Free tier services spin down after 15 minutes of inactivity (add monitoring to keep alive)
- For production, upgrade to Starter plan ($7/month) for always-on services
- Render provides free SSL certificates automatically
- Keep your Render services in the same region for optimal performance

Production Environment Variables:
----------------------------------
For all deployments, ensure these variables are set:
- NODE_ENV=production
- JWT_SECRET=your_very_secure_random_key (use a strong random string)
- MONGO_URI=your_mongodb_connection_string
- PORT=5000 (or as specified by hosting platform)
- FRONTEND_URL=https://yourdomain.com (for CORS configuration)
- BACKEND_URL=https://api.yourdomain.com (for frontend API calls)

MongoDB Setup:
--------------
- MongoDB Atlas (Recommended for production):
  1. Go to mongodb.com/cloud/atlas
  2. Create a cluster (free tier available)
  3. Get connection string: mongodb+srv://username:password@cluster.mongodb.net/dbname
  4. Use this as MONGO_URI in backend

- Self-hosted MongoDB:
  1. Install MongoDB on your server
  2. MONGO_URI: mongodb://username:password@hostname:27017/dbname

Performance Optimization for Production:
----------------------------------------
1. Enable gzip compression in backend (express-compression package)
2. Set appropriate cache headers for static assets
3. Use a CDN for frontend assets
4. Implement rate limiting on API endpoints (express-rate-limit)
5. Add HTTPS everywhere with proper SSL certificates
6. Monitor logs and set up error tracking (Sentry, LogRocket)
7. Use PM2 or similar process manager for Node.js with auto-restart
8. Implement database indexing for frequently queried fields

Security Checklist:
-------------------
[] JWT_SECRET is a strong random string (32+ characters)
[] MONGO_URI uses strong password authentication
[] CORS is properly configured for your domain
[] Environment variables are NOT committed to git
[] HTTPS is enabled for all connections
[] Rate limiting is enabled on auth endpoints
[] Input validation is implemented
[] Database backups are configured
[] Error messages don't expose sensitive information
[] Monitor for suspicious activity

Troubleshooting Deployment:
---------------------------
- Port already in use: Change PORT in .env or kill process using port
- MongoDB connection error: Verify MONGO_URI is correct and IP whitelist on MongoDB Atlas
- CORS errors: Update CORS origins in backend/server.js to match frontend domain
- Frontend routing 404: Ensure server redirects all routes to index.html
- Build fails: Clear node_modules and reinstall with npm install
- Memory issues: Increase container memory allocation or enable swap on server
